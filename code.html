一、 核心算法架构 (Core Architecture)

本项目最终采用 Minimax 算法 结合 Alpha-Beta 剪枝，并引入了 “杀手直觉” (Killer Heuristic) 预判机制，成功在浏览器端实现了 Depth=6 (六步算路) 的高性能 AI。

1. 决策流程图

  1. 第一层：紧急状态扫描 (Urgent Check)
    在进入耗时的递归搜索前，优先扫描棋盘是否存在“一步定胜负”的点（如己方连五、敌方活四）。
    if (checkUrgentMoves()) return move; // O(N) 复杂度，毫秒级响应
  2. 第二层：深度博弈搜索 (Deep Search)
    若无紧急情况，启动 6 层 Minimax 递归搜索，推演未来 3 个完整回合的局势。
  3. 第三层：启发式评估 (Evaluation)
    对叶子节点进行打分，结合“动态防御权重”计算最终落子价值。

三、 核心代码逻辑解析

1. 杀手直觉 (Killer Heuristic)

这是本项目最大的性能优化点。通过将“必胜/必救”逻辑从递归中剥离，避免了对显而易见结果的过度计算。

// 伪代码逻辑
function checkUrgentMoves() {
  if (我方有连五机会) return 进攻;
  if (敌方有活四/冲四威胁) return 防守;
  return null; // 进入慢速思考
}

2. 动态评分体系

为解决 AI “贪攻冒进”的问题，引入了基于威胁等级的动态权重。

  • 一般情况： 总分 = 进攻分 + (防守分 × 1.5)
  • 危急情况 (敌方分数 > 10000)： 总分 = 进攻分 + (防守分 × 2.0)
